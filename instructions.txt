== Benefits of this approach (development.json, as a separate file)

== How the config package works with environments?
The config package looks at your NODE_ENV environment variable.
Based on its value, it automatically loads the right JSON file from the config/ folder.

== Example Folder Structure
config/
   default.json
   development.json
   production.json
   test.json

== How it chooses the file?
_ If you don’t set anything → it uses 'default.json'.
_ If you set:
        export NODE_ENV=development
  it merges 'default.json' + 'development.json'.
_ If you set:
        export NODE_ENV=production
  it merges 'default.json' + 'production.json'
_ If you set:
        export NODE_ENV=test
  it merges 'default.json' + 'test.json'
